# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint was discovered through failure this session.

- [ ] CONSTRAINT: v-if unmounts components — Using `v-if` on a component that emits events while async work is in-flight will orphan all subsequent `emit()` calls. Use `v-show` instead to keep the component mounted.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| v-if orphaning emits | `UploadZone` was unmounted by `v-if="!isParsing"` when first progress event fired; all subsequent `emit('complete')` and `emit('progress')` were silently dropped | blocking | Always use `v-show` for components that emit events during long async operations; never `v-if` |
| output_config relay incompatibility | `output_config` JSON schema parameter caused silent empty response from relay; fell back to system prompt approach | advisory | When using third-party API relays, use system prompt for structured output, not newer SDK-specific params |
| SSE before regular POST | Frontend tried `/api/upload/stream` (404) before falling back to `/api/upload`; caused ECONNRESET errors | advisory | Only call endpoints that exist; SSE can be added later when backend implements it |
